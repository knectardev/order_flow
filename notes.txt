The information is all there and accurate— what is implemented the substantive feedback well. But you're right that readability is suffering. The tooltip has gone from undersharing to oversharing, and the visual hierarchy isn't doing enough work to help users parse it quickly.



**What's hurting readability:**

**Density without hierarchy.** Every line has the same visual weight. The eye has nowhere to land first, so it has to read everything to figure out what's important. Compare to a well-designed tooltip where the most important fact pops, supporting facts are secondary, and reference details are tertiary.

**Run-on measurement lines.** "Rejection: Exhaustion at upper wick · liquidity 0.39 · open ring (exhaustion · liquidity 0.39 · fill threshold 0.55) · moderate" is a single line containing five separate facts. The parenthetical actually repeats "exhaustion · liquidity 0.39" which is redundant. The eye can't process that in one pass.

**Self-referential meta-commentary in the body.** "Marker uses the side with the higher pipeline rejection score (vs the other extreme)." This is implementation explanation that probably doesn't belong in a hover tooltip—it's the kind of thing that goes in the legend or docs. Users hovering a candle want to know what the candle is showing, not how the rendering decision was made.

**The "Bands use visible bars only" footnote** is similar—it's a meta-explanation about how the encoding works, which clutters the analytical content.

**The interpretation sentence is buried at the bottom.** "Sell pressure held through a wick test at the extreme" is the plain-language gestalt and arguably the most useful single line for a quick read. It should be more prominent, not the second-to-last line.

**Suggested redesign:**

Here's a structure that preserves all the information but creates real visual hierarchy:

```
B-SHAPE · LOWER HALF · Wide body
"Sell pressure held through a wick test at the extreme"

Imbalance     1.51   strong (gate 0.30)
Delta         -925   matrix point red
Rejection     Upper wick · exhaustion · open ring
              Liquidity 0.39 (below 0.55 fill threshold) · moderate

PHAT READ IS DESCRIPTIVE, NOT PREDICTIVE
```

Three changes are doing the work:

1. **Lift the interpretation sentence to the top**, right under the header, in italic or quoted style. That's the gestalt; it should be the first thing the eye lands on after the title.

2. **Convert measurement lines to a label/value table format.** Aligned columns let the eye scan down the labels (Imbalance, Delta, Rejection) and then dip into the values when one matters. This is way easier to parse than prose-style "Imbalance: 1.51 (gate 0.30 · strong)".

3. **Drop the meta-commentary** ("Marker uses the side..." and "Bands use visible bars only..."). These belong in the legend, not in every hover tooltip. If users want to know how the encoding works, they'll consult the legend (which already has this info). The tooltip should focus on what *this specific candle* is telling them.

**For the neutral case specifically:**

The neutral tooltip has its own readability problem—the explanation of why it's neutral ("inside flow never cleared the gate for two-tone P/b shading") is doing the job of the gestalt sentence but in technical language. Suggest:

```
NEUTRAL · Below P/b gate · Wide body
"Auction tested one side at the wick; flow stayed mixed inside the bar"

Imbalance     0.04   below gate (0.30)
Delta         -439   matrix point red
Rejection     Lower wick · exhaustion · open ring
              Liquidity 0.34 (below 0.55 fill threshold) · moderate

PHAT READ IS DESCRIPTIVE, NOT PREDICTIVE
```

Same structure, just adapted to the neutral case. The interpretation line ("Auction tested one side at the wick; flow stayed mixed inside the bar") is plain-language and tells the user the analytical story; the measurements section gives them the numbers if they want them.

**On layout/typography:**

If Cursor can implement the label/value alignment, that's the biggest single readability win. Monospace already helps; explicit column alignment makes scanning trivial. If true table layout is too much, even just consistent indentation and bold labels would help:

```
Imbalance     1.51   strong (gate 0.30)
Delta         -925   matrix point red
```

Where "Imbalance" and "Delta" might be slightly muted/smaller and the values are normal weight. The eye learns to scan the label column, then jump to the value when interested.

**Optional: progressive disclosure.**

If the tooltip is still too dense even after this, consider a two-tier approach: hover shows the header + interpretation + key measurement (imbalance and delta only), and a "more" affordance reveals rejection details. That keeps the immediate hover lightweight while preserving access to the full data. But I'd try the restructure first before adding interaction complexity.
